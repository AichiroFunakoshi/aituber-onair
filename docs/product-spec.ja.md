# Product Spec

この文書は、このフォークを「自分のAITuberアプリ」として育てるための初期仕様です。
未確定項目は、実装前にここで決めてからコードへ反映します。

## 初期方針

- 作業名: Tuber
- リポジトリ: `AichiroFunakoshi/aituber-onair`
- 土台: `shinshin86/aituber-onair`
- 初期アバター方式: PNGTuber
- 初期配布形態: PWA
- 対象環境: macOS Safari、iOS Safari

PNGTuberを初期方式にする理由は、上流ドキュメントでも最初のローカルセットアップに
向いているとされており、VRM/Live2Dよりもアセット準備と検証負荷が小さいためです。

## 最初に作るもの

まずは、ブラウザで動く個人用AITuberアプリを作ります。

- キャラクターと会話できる。
- テキスト入力に返答できる。
- TTSで音声再生できる。
- 音声出力に合わせてPNGアバターの口が動く。
- macOS Safariで起動できる。
- iOS Safariで表示と音声再生を確認できる。

配信連携、YouTube/Twitchコメント対応、長期記憶、視聴者ポイント機能は、
最小アプリが安定してから段階的に追加します。

## 初期アプリ境界

共有パッケージを直接書き換える前に、既存サンプルまたはスターターをベースに
自分用アプリ領域を作ります。

候補:

1. `packages/core/examples/react-pngtuber-app` を検証用ベースにする。
2. `create-aituber-onair` の PNGTuber テンプレートを使い、モノレポ内に専用アプリを作る。
3. 仕様が固まったら `apps/tuber` のような専用領域へ分離する。

現在は `apps/tuber` を専用アプリ領域として使います。上流サンプルは参照用に残し、
自分のアプリ固有の変更は `apps/tuber` に集めます。

## 非目標

初期段階では以下をしません。

- SwiftUIで最初から作り直す。
- 上流の全パッケージを大きく改変する。
- App Store配布を前提にする。
- 複数LLM/TTSプロバイダを同時に最適化する。
- 秘密情報をリポジトリに入れる。
